Documentation & User Guidance

System Architecture
  • Frontend: HTML, CSS, JavaScript (Bootstrap)
  • Backend: Flask (Python)
  • ML Model: Support Vector Machine (SVM)
  • Data: CSV files for symptoms, disease info, and health suggestions
Functional Modules
  1. Symptom Input Module: Collects and validates user inputs
  2. Disease Prediction Engine: Uses SVM to predict disease from symptoms
  3. Recommendation Engine: Retrieves info on precautions, meds, diet, workout
  4. User Interface: Simple, responsive, and easy to use
ML Model Highlights
  • Trained using SVM algorithm
  • Works with symptom-based datasets
  • Optimized for balanced performance
Workflow Overview
User → Selects Symptoms
     → Clicks Submit
     → ML Model Predicts Disease
     → Backend Fetches Recommendations
     → Results Shown (Description, Meds, Diet, Workout)
